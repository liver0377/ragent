# Ragent Python 版 — 项目总览

## 1. 项目定位

Ragent 是一个 RAG（检索增强生成）智能体平台，Python 版在 Java 原版基础上做以下简化：

- 去除前端层，专注后端服务
- 去除 MCP-Server 独立进程，工具调用内聚到主应用
- 去除 Sa-Token 鉴权等企业级中间件
|- **保留核心**：Redis 缓存与分布式锁、消息队列异步处理、AI 模型路由、RAG 问答链路、文档入库流水线

||--------------|-----------------------------------------------|
|| 语言与框架   | Python 3.12+、FastAPI、Pydantic v2            |

|| SSE 流式推送 | FastAPI StreamingResponse                    |
|| 任务调度     | Celery + Redis（可选，用于后台入库任务）      |
|| 指标监控     | Prometheus + prometheus-client（指标采集）    |
|| 可视化面板   | Grafana（监控仪表盘、告警）                   |

|                     │   │

|                   │
|                   ▼
|    ┌────────────────────────────────┐
|    │         framework/             │
|    └────────────────────────────────┘
|```
|
|依赖原则：
|
|- **framework** 是最底层，不依赖任何业务模块，提供通用能力
|- **infra_ai** 依赖 framework，屏蔽模型供应商差异
|- **rag** 和 **ingestion** 依赖 infra_ai 和 framework
|- **app** 是顶层应用，依赖所有业务模块
