     1|# Ragent Python 版 — 项目总览
     2|
     3|## 1. 项目定位
     4|
     5|Ragent 是一个 RAG（检索增强生成）智能体平台，Python 版在 Java 原版基础上做以下简化：
     6|
     7|- 去除前端层，专注后端服务
     8|- 去除 MCP-Server 独立进程，工具调用内聚到主应用
     9|- 去除 Sa-Token 鉴权等企业级中间件
    10|- **保留核心**：Redis 缓存与分布式锁、消息队列异步处理、AI 模型路由、RAG 问答链路、文档入库流水线
    11|

    14|## 2. 技术栈
    15|
    16|| 层面         | 技术选型                                      |
    17||--------------|-----------------------------------------------|
    18|| 语言与框架   | Python 3.12+、FastAPI、Pydantic v2            |
    19|| 关系数据库   | PostgreSQL + SQLAlchemy 2.0（异步）           |
    20|| 向量数据库   | Milvus 2.x / pgvector（双引擎支持）           |
    21|| 缓存         | Redis 7.x + redis-py（异步）                  |
    22|| 任务队列     | Celery + Redis（异步任务分发，替代原版 RocketMQ）|
    23|| 文档解析     | python-docx / pdfplumber / unstructured       |
    24|| AI 模型调用  | litellm（统一多供应商接口）                   |
    25|| SSE 流式推送 | FastAPI StreamingResponse                    |
    26|| 任务调度     | Celery + Redis（可选，用于后台入库任务）      |
    27|| 指标监控     | Prometheus + prometheus-client（指标采集）    |
    28|| 可视化面板   | Grafana（监控仪表盘、告警）                   |

    67|## 4. 模块依赖关系
    68|
    69|```
    70|                  ┌─────────┐
    71|                  │  app/   │
    72|                  └──┬───┬──┘
    73|                     │   │
    74|          ┌──────────┘   └──────────┐
    75|          ▼                         ▼
    76|    ┌───────────┐           ┌────────────┐
    77|    │   rag/    │           │ ingestion/ │
    78|    └─────┬─────┘           └──────┬─────┘
    79|          │                        │
    80|          ▼                        ▼
    81|    ┌────────────────────────────────┐
    82|    │          infra_ai/             │
    83|    └──────────────┬─────────────────┘
    84|                   │
    85|                   ▼
    86|    ┌────────────────────────────────┐
    87|    │         framework/             │
    88|    └────────────────────────────────┘
    89|```
    90|
    91|依赖原则：
    92|
    93|- **framework** 是最底层，不依赖任何业务模块，提供通用能力
    94|- **infra_ai** 依赖 framework，屏蔽模型供应商差异
    95|- **rag** 和 **ingestion** 依赖 infra_ai 和 framework
    96|- **app** 是顶层应用，依赖所有业务模块

   100|## 5. 核心业务流程概览
   101|
   102|一次用户提问经过的完整链路：
   103|
   104|```
   105|用户提问
   106|  │
   107|  ▼
   108|┌─────────────────────────────────────────┐
   109|│ 1. 限流排队                              │
   110|│    Redis ZSET + Lua 排队 / 并发控制       │
   111|└─────────────────┬───────────────────────┘
   112|                  ▼
   113|┌─────────────────────────────────────────┐
   114|│ 2. 问题重写                              │
   115|│    多轮上下文补全 / 复杂问题拆分          │
   116|└─────────────────┬───────────────────────┘
   117|                  ▼
   118|┌─────────────────────────────────────────┐
   119|│ 3. 意图分类                              │
   120|│    LLM 打分 → 置信度过滤 → 路由决策      │
   121|└──────┬──────────────────┬───────────────┘
   122|       ▼                  ▼
   123|┌──────────────┐  ┌──────────────┐
   124|│ 4a. 多路检索  │  │ 4b. 工具调用  │
   125|│ 向量检索+重排 │  │ 内置工具执行  │
   126|└──────┬───────┘  └──────┬───────┘
   127|       │                  │
   128|       ▼                  ▼
   129|┌─────────────────────────────────────────┐
   130|│ 5. Prompt 组装                           │
   131|│ 系统提示 + 检索上下文 + 对话历史 + 问题   │
   132|└─────────────────┬───────────────────────┘
   133|                  ▼
   134|┌─────────────────────────────────────────┐
   135|│ 6. 模型生成（SSE 流式输出）               │
   136|│ 多候选路由 → 首包探测 → 自动降级          │
   137|└─────────────────┬───────────────────────┘
   138|                  ▼
   139|┌─────────────────────────────────────────┐
   140|│ 7. 后处理                                │
   141|│ 消息持久化 / 会话记忆更新 / 链路追踪      │
   142|└─────────────────────────────────────────┘
   143|```
