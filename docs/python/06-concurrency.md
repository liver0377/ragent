# Ragent Python 版 — 并发模型与限流排队

## 1. 并发模型

Python 版不使用 Java 的多线程池模型，而是基于 asyncio 协程实现并发。协程天生适合 I/O 密集型场景（模型调用、数据库查询、Redis 操作），且开销远低于线程。

---

```
│  │                                                      │ │
│  │  ├── HTTP 请求处理（协程）                            │ │
│  │  ├── SSE 流式推送（异步生成器）                       │ │
│  │  ├── MQ 消费者（aio-pika 协程）                      │ │
│  │  ├── Redis 操作（redis-py async）                    │ │
│  │  ├── 数据库操作（SQLAlchemy async）                   │ │
│  │  └── 外部模型调用（httpx async）                     │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                           │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  线程池执行器 (可选)                                   │ │

└──────────────────────────────────────────────────────────┘
```

## 3. 限流排队机制

### 3.1 整体流程

```
用户请求
   │
   ▼
┌────────────────────────┐
│  Redis ZSET 排队        │
│  请求入队，按时间戳排序  │
└───────────┬────────────┘
            │
            ▼
┌────────────────────────┐
│  Lua 脚本原子判断       │
│  是否在队头窗口内        │
└─────┬──────────┬───────┘
      │          │
   允许执行    排队等待
      │          │
      │          ▼
      │   ┌────────────────────┐
      │   │  Redis Pub/Sub 等待 │
      │   │  监听释放通知        │
      │   └────────┬───────────┘
      │            │
      │            ▼
      │   ┌────────────────────┐
      │   │  SSE 推送排队状态    │
      │   │  实时告知用户位置    │
      │   └────────┬───────────┘
      │            │
      ▼            ▼
┌────────────────────────┐
│  Semaphore 并发控制      │
│  限制同时执行的请求数    │
│  许可带超时自动释放      │
└───────────┬────────────┘
            │
            ▼
┌────────────────────────┐
│  执行 RAG 问答链路      │
└───────────┬────────────┘
            │
            ▼
┌────────────────────────┐
│  Redis Pub/Sub 通知     │
│  唤醒下一个排队请求      │
└────────────────────────┘
```

### 3.2 Redis ZSET 排队详解

```
┌──────────────────────────────────────────────────┐
│  Redis ZSET: ragent:queue:{user_id}               │
│                                                   │
│  Score (时间戳)     Member (请求ID)               │
│  1700000001.001    req_abc123                     │
│  1700000002.500    req_def456                     │
│  1700000003.100    req_ghi789                     │
│                                                   │
│  Lua 脚本逻辑:                                    │
│  1. 查询当前请求的排名                             │
│  2. 排名 <= max_concurrent → 允许执行              │
│  3. 排名 >  max_concurrent → 返回排队位置          │
│  4. 请求完成后 ZREM 移除                          │
│  5. PUBLISH 通知下一个等待者                       │
└──────────────────────────────────────────────────┘
```

### 3.3 Semaphore 并发控制

```
┌───────────────────────────────────────────────────┐
│  asyncio.Semaphore (max_concurrent)                │
│                                                    │
│  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐       │
│  │ 1 │ 2 │ 3 │ 4 │ 5 │   │   │   │   │   │       │
│  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘       │
│    使用中                    空闲许可               │
│                                                    │
│  特性:                                             │
│  ├── 获取许可时带超时，防止死锁                     │
│  ├── 使用 async with 自动释放                      │
│  └── 协程级别控制，无锁开销                         │
└───────────────────────────────────────────────────┘
```

## 5. 并行检索调度

多路检索使用 asyncio.gather 实现并行：

```
重写后的问题
     │
     ├── asyncio.gather() ──┐
     │                      │
     │         ┌────────────┼────────────┐
     │         ▼            ▼            ▼
     │  ┌───────────┐ ┌──────────┐ ┌───────────┐
     │  │意图定向检索│ │全局向量检索│ │(可扩展...) │
     │  │协程        │ │协程       │ │            │
     │  └─────┬─────┘ └────┬─────┘ └─────┬─────┘
     │        │             │             │
     │        └─────────────┼─────────────┘
     │                      │
     │                      ▼
     │            ┌─────────────────┐
     │            │   结果合并       │
     │            └────────┬────────┘
     │                     ▼
     │            ┌─────────────────┐
     │            │  去重后处理器     │
     │            │ (跨通道去重)     │
     │            └────────┬────────┘
     │                     ▼
     │            ┌─────────────────┐
     │            │  重排序后处理器   │
     │            │ (Rerank模型)    │
     │            └────────┬────────┘
     │                     ▼
     │               最终检索结果
```

每个检索通道是独立的协程，通过 asyncio.gather 并行调度，互不阻塞。后处理器按顺序串联执行。

## 6. 超时与降级

```
┌──────────────────────────────────────────────────┐
│  超时层次                                         │
│                                                   │
│  ├── 请求级超时    → 整个 RAG 链路最长等待时间     │
│  │   (asyncio.wait_for, 120s)                     │
│  │                                                │
│  ├── 检索级超时    → 单次检索最长等待时间          │
│  │   (asyncio.wait_for, 30s)                      │
│  │                                                │
│  ├── 模型级超时    → 单次模型调用最长等待时间      │
│  │   (httpx timeout, 60s)                         │
│  │                                                │
│  └── 首包超时      → 流式输出等待首包最长时间      │
│      (asyncio.wait_for, 60s)                      │
│                                                   │
│  降级策略:                                        │
│  ├── 检索超时 → 返回已有结果，跳过超时通道         │
│  ├── 模型超时 → 自动切换到下一个候选模型           │
│  └── 全部超时 → 返回兜底提示 + 记录 Trace          │
└──────────────────────────────────────────────────┘
```

## 7. 与 Java 版的关键差异

| 维度         | Java 版                      | Python 版                      |
|-------------|------------------------------|--------------------------------|
| 并发模型     | 8 个独立线程池                | asyncio 协程 + 可选线程池      |
| 上下文透传   | TransmittableThreadLocal      | ContextVar (协程原生)          |
| 限流实现     | Redisson Semaphore            | asyncio.Semaphore + Redis Lua  |
| 状态通知     | Redis Pub/Sub + 本地合并      | Redis Pub/Sub (协程订阅)       |
| SSE 推送     | SseEmitter + CAS 线程安全     | FastAPI StreamingResponse      |
| 并行调度     | ThreadPoolExecutor            | asyncio.gather                 |
| CPU 任务     | ForkJoinPool                  | ProcessPoolExecutor (可选)     |
